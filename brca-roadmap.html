PHASE 01

Environment & Dataset Setup

Get your tools and data ready before writing a line of model code
~1 week ▾

Goal: Have a clean, reproducible project structure with all three datasets downloaded, understood, and committed to version control.

1.1 — Set up your Python environment
Create a virtual environment and install core dependencies. Use Python 3.10+. Pin all package versions in a requirements.txt from the start — this saves pain later when sharing or submitting.
# Terminal python -m venv brca-env source brca-env/bin/activate pip install torch scikit-learn pandas numpy biopython shap matplotlib seaborn jupyter
1.2 — Download ClinVar BRCA variants
Go to ncbi.nlm.nih.gov/clinvar and filter by gene: BRCA1 and BRCA2. Download the VCF or TSV file. You want the "clinical significance" column — this is your label (Pathogenic vs. Benign/Likely Benign). Expect ~20,000–50,000 labeled variants.
1.3 — Access TCGA-BRCA clinical data
Register at the GDC portal (portal.gdc.cancer.gov). Download the TCGA-BRCA clinical manifest. You want: age at diagnosis, tumor stage, ER/PR/HER2 status, family history flags, and mutation annotations. Use the GDC client tool for bulk download.
1.4 — Pull reference BRCA sequences from NCBI
Use Biopython's Entrez module to fetch the canonical BRCA1 and BRCA2 mRNA/coding sequences. These become your reference for constructing variant sequences in Phase 2.
from Bio import Entrez, SeqIO Entrez.email = "your@email.com" handle = Entrez.efetch(db="nucleotide", id="NM_007294", rettype="gb") # NM_007294 = BRCA1, NM_000059 = BRCA2
1.5 — Create a structured project repo
Initialize a GitHub repo with folders: /data/raw, /data/processed, /notebooks, /src, /models, /results. Add a .gitignore for large data files. Write a README.md now — even just a skeleton. This discipline pays off.
✓ Phase 1 checkpoint: You can load all three data sources into a Jupyter notebook and print their shape/columns. Your repo is on GitHub with a clean folder structure.

PHASE 02

Feature Engineering

Convert raw DNA sequences and clinical records into model-ready tensors
~1.5 weeks ▾

Goal: Produce two clean feature matrices — one for sequence embeddings, one for clinical features — that can be merged on patient/variant IDs.

2.1 — Build k-mer frequency embeddings from variant sequences
For each variant in ClinVar, reconstruct the local sequence context (~100bp window around the mutation position). Slide a window of k=3,4,5 over the sequence and count each k-mer. This gives you a fixed-length vector per variant without needing alignment. Normalize using TF-IDF style weighting.
from collections import Counter def kmer_freqs(seq, k=4): kmers = [seq[i:i+k] for i in range(len(seq)-k+1)] counts = Counter(kmers) return counts # normalize after
2.2 — Encode mutation type features
Beyond k-mers, extract explicit mutation-level features: variant type (SNV, insertion, deletion, frameshift), position in the gene (exon number, distance from splice site), GC content of the surrounding region, and conservation score if available. These are cheap to compute and highly informative.
2.3 — Process clinical tabular features
From TCGA-BRCA: select age, tumor grade (I–III), ER/PR/HER2 status (one-hot), and any BRCA family history flag. Handle missing values with median imputation for continuous features, and "unknown" category for categorical. Standardize continuous features with StandardScaler.
2.4 — Merge datasets and define train/val/test splits
Join sequence features and clinical features on shared identifiers. Use an 70/15/15 stratified split, stratifying on the pathogenicity label to ensure balanced classes in each split. Save the splits to disk so they're fixed and reproducible — do not re-split during training.
2.5 — Exploratory data analysis
Before modeling, spend time here. Plot class balance (pathogenic vs benign). Visualize top k-mers by class using bar charts. Create a correlation heatmap of clinical features. Check for data leakage vectors. Document your findings in a notebook — this section often becomes the Methods section of a paper.
✓ Phase 2 checkpoint: You have X_seq, X_clinical, and y arrays saved as .npy files with confirmed train/val/test splits. EDA notebook is clean and committed.

PHASE 03

Baseline Models

Establish strong baselines before building the fusion architecture
~1 week ▾

Goal: Measure how well sequence-only and clinical-only models perform separately. This sets the bar your fusion model must beat — and is essential for any paper.

3.1 — Train a sequence-only baseline (Random Forest)
Feed only the k-mer frequency vectors into a Random Forest classifier. Tune n_estimators and max_depth with cross-validation. This is your sequence-only ceiling. Record AUC-ROC, precision, recall, and F1 on the validation set.
3.2 — Train a clinical-only baseline (XGBoost)
Use only the clinical tabular features with XGBoost. XGBoost handles missing values natively and typically outperforms RF on tabular data. Tune learning_rate, max_depth, and n_estimators. Log all metrics identically to 3.1.
3.3 — Simple concatenation baseline (neural net)
Concatenate X_seq and X_clinical into one flat vector and train a 2-layer MLP. This is the naive fusion approach — your final model should beat this by using a proper fusion architecture rather than just concatenation.
# Simple MLP baseline model = nn.Sequential( nn.Linear(seq_dim + clin_dim, 256), nn.ReLU(), nn.Dropout(0.3), nn.Linear(256, 64), nn.ReLU(), nn.Linear(64, 1), nn.Sigmoid() )
3.4 — Build a results tracking table
Create a spreadsheet (or use MLflow/Weights & Biases if you want to be rigorous) logging each model, its hyperparameters, and val set metrics. Every future run goes in here. This table becomes Table 1 in your paper.
✓ Phase 3 checkpoint: You have 3 baseline models with recorded AUC-ROC scores. You know which input modality is more predictive on its own.

PHASE 04

Fusion Model Architecture

Build the multimodal model that is the core contribution
~2 weeks ▾

Goal: Design and train a model with separate encoders for each modality that fuses representations via a learned attention mechanism — not just concatenation.

4.1 — Build the sequence encoder branch
Design a small 1D-CNN or MLP that takes the k-mer vector and outputs a 128-dim embedding. The key is that this branch learns a compact representation of the sequence context before fusion, rather than dumping raw features into a combined layer.
class SeqEncoder(nn.Module): def __init__(self, input_dim): super().__init__() self.net = nn.Sequential( nn.Linear(input_dim, 256), nn.GELU(), nn.LayerNorm(256), nn.Dropout(0.2), nn.Linear(256, 128) # 128-dim embedding )
4.2 — Build the clinical encoder branch
Similarly, build a separate MLP encoder for clinical features that also outputs a 128-dim embedding. Using the same output dimension makes the fusion step symmetric and cleaner. Add batch normalization since clinical features have very different scales.
4.3 — Design the cross-modal fusion layer
This is the novel part. Instead of concatenating, use a cross-attention mechanism: let the clinical embedding "attend" to the sequence embedding and vice versa. Stack both into a 2-token sequence and apply a single transformer attention head. The model learns which modality to trust more per sample.
# Stack embeddings as 2 tokens, apply attention tokens = torch.stack([seq_emb, clin_emb], dim=1) # [B, 2, 128] attended = self.attn(tokens, tokens, tokens) # self-attention fused = attended.mean(dim=1) # [B, 128]
4.4 — Add the classification head and train
Pass the fused 128-dim vector through a 2-layer MLP to a sigmoid output. Train with BCELoss, AdamW optimizer, and a cosine learning rate scheduler. Use early stopping on val AUC with patience=10. Address class imbalance using pos_weight in BCELoss.
4.5 — Ablation study
Train three variants: seq-only encoder, clinical-only encoder, and full fusion. This directly proves the value of multimodal fusion and gives you a clean narrative: each modality alone is weaker, together they're stronger. Essential for any submission.
✓ Phase 4 checkpoint: Your fusion model beats all three Phase 3 baselines on validation AUC-ROC. Ablation table shows the value of each modality. Model is saved to disk.

PHASE 05

Explainability Layer

Make the model interpretable — the part most papers skip
~1.5 weeks ▾

Goal: Produce per-patient and global explanations that show which features (specific k-mers, clinical variables, mutation positions) drove each prediction.

5.1 — Apply SHAP DeepExplainer to the fusion model
Use SHAP's DeepExplainer (designed for PyTorch networks) on your test set. This assigns each input feature a SHAP value — its marginal contribution to the prediction. Run it on a background dataset of ~100 samples for efficiency on your laptop/GPU.
import shap explainer = shap.DeepExplainer(model, background_data) shap_values = explainer.shap_values(X_test)
5.2 — Global feature importance: top k-mers and clinical features
Plot mean absolute SHAP values across all test samples for both modalities. Which k-mer sequences most strongly predict pathogenicity? Which clinical features dominate? This is a publishable figure — you may find biologically meaningful patterns, e.g., specific trinucleotide contexts.
5.3 — Per-patient explanation waterfall plots
Generate waterfall plots for 3–5 representative patients: one true positive, one true negative, one false positive. Show visually why the model made each call. This builds trust and is extremely valuable for any clinical audience reading your paper.
5.4 — Visualize attention weights from the fusion layer
For each test sample, extract the cross-attention weights. Plot the distribution: when does the model rely more on sequence vs clinical features? If BRCA1 variants show higher sequence attention than BRCA2 variants, that's a novel biological finding worth discussing.
✓ Phase 5 checkpoint: You have a global SHAP summary plot, 3 waterfall plots, and an attention weight analysis. These become Figures 2–4 in your paper.

PHASE 06

Evaluation, Write-up & Publication

Finalize results, write the paper, and share your work
~2 weeks ▾

Goal: Produce a clean, reproducible codebase and a paper-ready write-up for arXiv or a bioinformatics journal submission.

6.1 — Final evaluation on held-out test set
Run your best fusion model on the test set exactly once. Report AUC-ROC, AUPRC (area under precision-recall curve — more informative for imbalanced data), F1, sensitivity, and specificity. Run bootstrap resampling (n=1000) to get 95% confidence intervals on all metrics.
6.2 — Stretch goal: fine-tune ESM-2 sequence encoder
If time allows, swap your k-mer encoder for Meta's ESM-2 protein language model (available on HuggingFace). Fine-tune the last 2 layers on your BRCA variant sequences. This typically boosts sequence representation quality significantly and makes the project more publishable at top venues.
6.3 — Clean and document the codebase
Add docstrings, a detailed README with setup instructions, a data download script, and a single train.py entry point. The goal: a stranger should be able to clone and reproduce your results in under an hour. This dramatically increases citations if you publish.
6.4 — Write the paper (or technical report)
Target arXiv first, then consider Bioinformatics (Oxford), PLOS Computational Biology, or Briefings in Bioinformatics. Structure: Abstract → Introduction → Related Work → Methods → Results → Discussion → Conclusion. Your ablation table, SHAP plots, and attention analysis are ready-made figures.
6.5 — Release and share
Post to arXiv, share the GitHub repo publicly, and post a summary thread on X/Twitter or LinkedIn tagging relevant ML and genomics communities. Consider submitting a demo to Hugging Face Spaces. Projects that are easy to find and run get cited; dark repos don't.
✓ Phase 6 checkpoint: arXiv preprint is live, GitHub repo is public with a star-worthy README, and you have a complete results table with confidence intervals.
